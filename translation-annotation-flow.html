🏗️ Translation & Annotation Architecture

共享基础设施，差异化UI渲染

数据流程
✅ 完全共享
⚠️ 部分共享或差异化
📝 Stage 1: 用户输入

翻译功能

• 用户选中文本
• 点击 "T" 按钮
• 触发 translateText()

注释功能

• 用户选中文本
• 点击 "A" 按钮
• 触发 annotateSelectedText()
⬇️
🔄 Stage 2: 翻译服务 100% 共享

TranslationService.translate()

两个功能都调用同一个方法：
• 参数相同：text, targetLanguage, sourceLang
• 缓存策略相同
• 提供商选择相同
• 返回数据结构相同
⬇️
💾 Stage 3: 缓存检查 100% 共享

统一缓存机制

• Cache Key: ${text}:${sourceLang}:${targetLang}:${provider}
• LRU 策略（最近最少使用）
• 可配置大小（10-1000 项）
• 翻译和注释共享同一个缓存池
⬇️
🌐 Stage 4: 翻译提供商 100% 共享

Debug Provider

• 返回测试数据
• 包含完整的音标、词义、例句
• 支持 annotationText 生成

Youdao Provider

• 调用有道 API
• 解析音标、词义、例句
• 音标补充机制（FreeDictionary）

Google Provider

• 调用 Google Translate
• 基础翻译功能
• 可扩展音标支持
⬇️
📦 Stage 5: 数据结构 100% 共享

{
  originalText: "technique",

  sourceLang: "en",

⬇️
🎨 Stage 6: UI 渲染 差异化设计

翻译功能 UI

TranslationUI.render(result)

✅ 原文（originalText）
✅ 译文（translatedText）
✅ 读音（phonetics[]）- 完整列表
✅ 词义（definitions[]）- 全部显示
✅ 例句（examples[]）- 全部显示
✅ 提供商（provider）
✅ 关闭按钮

形式：悬浮卡片（Tooltip）

注释功能 UI

createRubyAnnotation(result)

✅ 标注文本（annotationText）
✅ 音频按钮（如果有 phonetics）
❌ 不显示词义（空间限制）
❌ 不显示例句（空间限制）

形式：内联 Ruby 标签
<ruby>technique<rt>/tekˈniːk/ 技术 🔊</rt></ruby>
⬇️
🔊 Stage 7: 音频播放 70% 共享

共享部分

✅ 相同的 phonetics 数据源
✅ 相同的音频 URL/数据
✅ 都支持 Web Speech API（TTS）
✅ 都使用 LRU 缓存策略

实现差异

⚠️ 翻译：TranslationUI 内的实现
⚠️ 注释：content.js 内的独立实现
⚠️ 各自维护独立的音频缓存

可优化：统一到共享的 AudioService
