🔧 翻译提供商选择 - 换行问题修复

❌ 问题描述

在 Options 页面中，翻译提供商选择的三个单选按钮出现了文字换行的问题。 "谷歌翻译"、"有道翻译"、"DeepL翻译" 等文字在按钮内换行显示，影响美观和用户体验。

原因：原 min-width: 150px 太小，无法容纳图标 (20px) + 间距 (8px) + 中文文字 + 左右内边距 (24px)

🔍 修复前后对比

修复前

  • 文字换行，显示不整齐
  • min-width: 150px 太小
  • 缺少 white-space: nowrap
  • 内边距偏小（12px）

修复后

  • 文字单行显示，整齐美观
  • min-width: 200px 增加宽度
  • 添加 white-space: nowrap
  • 优化内边距（12px 16px）

💻 修复的代码变更

1. .radio-item 样式调整

.radio-item { flex: 1; min-width: 150px; /* 修复前：太小 */ min-width: 200px; /* 修复后：增加到 200px */ padding: 12px; /* 修复前 */ padding: 12px 16px; /* 修复后：增加水平内边距 */ background: #f8f9fa; border: 2px solid #e0e0e0; border-radius: 6px; cursor: pointer; transition: all 0.2s; display: flex; align-items: center; white-space: nowrap; /* 新增：防止文字换行 */ }

2. .provider-option-label 样式调整

.provider-option-label { display: flex; align-items: center; gap: 8px; white-space: nowrap; /* 新增：确保标签不换行 */ }

3. .provider-option-logo 优化

.provider-option-logo { width: 20px; height: 20px; object-fit: contain; vertical-align: middle; flex-shrink: 0; /* 建议添加：防止图标被压缩 */ }

📝 修复要点总结

属性 修复前 修复后 说明
min-width 150px 200px 增加最小宽度，确保文字不换行
padding 12px 12px 16px 增加水平内边距，视觉更舒适
white-space - nowrap 强制单行显示，防止换行
flex-shrink - 0 防止图标被压缩（建议）

📱 响应式设计保持不变

在小屏幕（< 600px）上，选项会自动切换为纵向排列，确保每个选项占据 100% 宽度：

@media (max-width: 600px) { .radio-group { flex-direction: column; } .radio-item { min-width: 100%; } }
  • 移动端自动纵向排列
  • 每个选项占满宽度
  • 保持良好的触摸体验

✅ 测试验证

  • 在不同浏览器中测试显示效果（Chrome、Firefox、Safari）
  • 调整浏览器窗口宽度，验证不同尺寸下的显示
  • 确认中英文文字都能正常单行显示
  • 验证三个选项在同一行均匀分布
  • 检查小屏幕（< 600px）下的纵向排列
