# Architecture Decision Records

This section records important architectural decisions made during the development of Annotate Translate.

Each ADR documents the context, options considered, decision made, and consequences.

| ADR | Title | Status | Date |

## How to Create an ADR

1. Copy `000-template.md` to a new file with the next sequential number (e.g., `001-decision-title.md`)
2. Fill in the context, options, decision, and consequences
3. Set status to `proposed`
4. After team review, update status to `accepted`

## Status Definitions

| Status | Meaning |
|--------|---------|
| proposed | Under discussion, not yet decided |
| accepted | Decision made, implementation should follow |
| deprecated | No longer relevant, superseded by another decision |
| superseded | Replaced by a newer ADR (link in `superseded-by` field) |
